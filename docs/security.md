## 管理员认证

- 密码使用 `bcrypt` 算法哈希存储，避免明文保存。
- 首次启动时若 `users` 表为空：
  - 使用 `ADMIN_USERNAME`（默认 `admin`）作为用户名。
  - 若 `ADMIN_PASSWORD` 未设置，后端使用 `crypto/rand` 生成 16 位随机密码，并打印到标准输出。
- 登录时通过 `bcrypt.CompareHashAndPassword` 校验密码。
- 仅存在一个内置管理员账号，当前未提供多管理员管理接口。

## JWT 设计

- 算法：`HS256`。
- 签名密钥来自环境变量 `JWT_SECRET`；若未设置，启动时生成随机 32 字节 hex 字符串。
- Token 声明：
  - `sub`：用户名。
  - `jti`：唯一 token ID，用于黑名单登出。
  - `iat`：签发时间。
  - `exp`：过期时间，由 `JWT_EXPIRE_HOURS` 控制（默认 24 小时）。
- 登出时解析 token，提取 `jti` 和剩余有效期，写入 Redis 黑名单 `jwt:blacklist:<jti>`，TTL 与剩余有效期一致。
- 认证中间件校验签名、过期时间和黑名单状态。

> 警告：如果 `JWT_SECRET` 为空且未持久化，每次重启后端都会生成新的随机密钥，导致所有已签发 token 失效。生产环境务必显式设置 `JWT_SECRET`。

## 会话安全

- 前端将 JWT 保存在 `localStorage`。
- Axios 请求拦截器自动附加 `Authorization: Bearer <token>`。
- 响应拦截器检测到 `401` 时清除 token 并跳转登录页。
- 生产环境建议：
  - 启用全站 HTTPS，防止 token 被中间人截获。
  - 设置合理的 `JWT_EXPIRE_HOURS`，缩短 token 有效期。
  - 管理后台不使用 `localStorage` 之外的持久化存储 token。
  - 考虑为管理后台站点配置 CSP、HSTS 等安全响应头。

## 接口权限控制

- `/api/admin/*` 全部需要 JWT 认证（除 `/api/admin/login` 外）。
- 代理路由不需要认证，对外提供服务。
- 当前未实现基于角色的细粒度权限控制，仅区分“已登录管理员”和“未登录用户”。

## 输入校验

- 使用 Gin 的 `binding` 标签对请求体进行必填、类型、URL 格式校验。
- 域名与目标地址在保存前统一转小写、去除前后空格。
- JSON 字段映射规则在保存时会校验是否为合法 JSON 数组。
- 请求体大小限制为 10MB，防止过大请求消耗资源。

## 风控隔离

- 风控规则仅影响匹配的路径与方法，避免误伤其他接口。
- Redis 计数 key 包含规则 ID 与身份标识，格式为 `attempt:<rule_id>:<identity>`，防止跨规则污染。
- 身份标识可以选择仅按 IP，或按 `IP + JSON 请求体字段` 组合，灵活平衡安全与用户体验。
- `duplicate_ip` 与 `rate_limit` 两种规则类型分别适用于不同场景，避免单一策略导致过度拦截。

## 日志与敏感数据

- 所有代理请求和拦截事件写入 `proxy_logs`。
- 记录内容包含：
  - 始终记录：绑定域名、客户端 IP、方法、路径、目标地址、状态码、是否拦截、User-Agent、请求体原文。
  - 拦截时额外记录：查询参数、请求头。
- 当前实现会记录所有请求的请求体原文，如果上游传输敏感信息（如密码、身份证号、银行卡号），存在数据泄露风险。

### 建议

- 对敏感字段进行脱敏后再记录，或在数据库层设置更短的数据保留周期。
- 生产环境定期清理或归档 `proxy_logs`。
- 限制数据库访问权限，避免日志被非授权人员读取。

## 生产安全建议

| 项目 | 建议 |
|------|------|
| 管理员密码 | 使用强密码，定期更换，不要保留 `.env.example` 默认值。 |
| JWT 密钥 | 显式设置 `JWT_SECRET`，长度不低于 32 位随机字符。 |
| 网络暴露 | 后端只监听 `127.0.0.1`，通过 Nginx/1Panel 反向代理。 |
| 管理后台访问 | 限制来源 IP 或仅内网可达。 |
| HTTPS | 强制 HTTPS，申请 SSL 证书。 |
| 数据库密码 | 修改 PostgreSQL 和 Redis 默认密码。 |
| 日志 | 配置日志大小限制，定期清理敏感日志。 |
| 升级 | 及时更新基础镜像和依赖，修复已知漏洞。 |

## 已知限制

- 当前仅支持单管理员账号，无法通过界面新增/删除管理员。
- `JWT_SECRET` 未设置时会随机生成，重启后所有 token 失效。
- `proxy_logs.request_body` 记录所有请求体，未做敏感字段脱敏。
- `Rule.BlockSeconds` 字段已预留但未实现拦截后额外封禁逻辑。
- `Domain.ResponseTransform` 字段已预留但未对上游响应生效。
